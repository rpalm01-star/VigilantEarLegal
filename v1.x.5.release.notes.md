# VigilantEar Release Notes — Branch: `chore/android-begins`

This release represents a major milestone: the official birth and core parity implementation of the **Android version** of VigilantEar, along with major structural synchronization, test automation, and privacy-first updates on the **iOS/Apple side**.

---

## 👥 User-Facing Release Notes

### 🤖 Android Version Debut (Beta 1.1.5)
VigilantEar is now officially on Android! Built from the ground up to achieve feature and design parity with the iOS version.
*   **Tactical Sonar Radar & HUD:** Features the tactical dark-themed dashboard. Provides real-time updates on active acoustic targets, estimated bearings, signal energy, and system engine telemetry.
*   **Google Maps Integration:** Tracks vehicles and sirens in real time on a customized high-contrast dark satellite map.
*   **Centralized Startup Verification:** Integrates the safety gate checklist flow. It checks microphone permissions, location services, and neural processor availability concurrently upon launch, ensuring the app is armed and safe to monitor.
*   **Severe Weather Alerts (CAP):** Keeps you safe by monitoring official emergency weather feeds (National Weather Service in the US, MeteoGate in Europe, and CMA/MEM in China) and displaying active warning zones on the map.
*   **Google Play Billing Support:** Full in-app purchase and subscription verification to match iOS Premium tiers.
*   **Full Localization Matrix:** Native support for English, Spanish, French, German, Japanese, Portuguese, and Simplified Chinese.

### 🍎 Apple Version Updates
*   **Improved Startup Flow:** Refined the Permission and Startup verification screens with modern layout scaling, title bloom animations, and a smoother UX when returning from system Settings.
*   **ShazamKit Improvements:** Upgraded song detection to instantly map real-time music metadata directly onto the sonar grid when high-confidence music is heard.
*   **Privacy-First Operation:** Stripped out third-party crash and analytical telemetry trackers, ensuring all acoustic signal evaluations and logs remain 100% local to your device hardware.
*   **Dynamic Thermal Safety & HUD Status:** Integrated device thermal state monitoring via Apple's `ProcessInfo.ThermalState`. The tactical HUD radar now reports live temperature telemetry (NOMINAL, WARM, ELEVATED, CRITICAL). If the device reaches SERIOUS or CRITICAL thermal thresholds, VigilantEar automatically throttles back its machine learning frame analysis duty cycle (to 37.5% and 12.5% respectively) and triggers a warning banner, protecting battery and processor life while continuing active audio recording.

## 🛠️ Technical & Engineering Release Notes (Internal)

### 1. Android NDK & C++ Acoustic Engine
*   **Oboe Audio Pipeline:** Replaced Apple's AVAudioEngine with the C++ **Oboe** library for low-latency, real-time PCM audio stream capture.
*   **JNI Instance Isolation:** Refactored JNI functions to bind to a native pointer (`nativePtr`) mapping. This decouples class instances from global state and eliminates race conditions on dual-channel callback streams.
*   **Dual-Classifier Audio Pipeline:** Built a primary classifier (using MediaPipe + YAMNet) delegated to the mobile NPU for critical emergency triggers, and a secondary CPU-delegated neural ticker for continuous ambient classifications. 
*   **Acute vs. Broadband Trigger Logic:**
    *   *Acute sounds* (alarms, glass breaking) trigger instantly via Peak (+16dB) and RMS (+3.5dB) spectral thresholds.
    *   *Broadband sounds* (vehicles, music) utilize lower confidence seeding (`0.10`) and temporal grouping to avoid dropouts.
*   **TDOA Spatial Math:** Integrated `kiss_fft` and Time Difference of Arrival (TDOA) phase coherence algorithms in native C++ to compute directional bearings from stereo microphone hardware signals.

### 2. Physics & Navigation Port
*   **VehiclePathPredictor:** Ported iOS Doppler tracking and path-planning algorithms to Kotlin. Snaps relative acoustic bearings to Google Maps GPS coordinates and predicts vehicle trajectories.
*   **CAP Alert Fetching:** Constructed Kotlin coroutine managers to fetch and parse XML/GeoJSON emergency weather alerts via secure REST clients.

### 3. iOS Test & CI Refactoring
*   **Headless Test Execution:** Decoupled `TEST_HOST` requirements in the Xcode unit test targets. Unit tests for mathematical and physics packages (like `FFTProcessorTests` and `SongSignatureTests`) can now run directly without launching the full iOS app host, saving pipeline run times.
*   **GitHub Actions CI/CD:** Upgraded the Android and iOS CI runners to Node.js 24 and Ubuntu, cache-linked the Android NDK, and configured automated simulator boots for clean Xcode unit test verification.

### 4. Codebase-wide Privacy & Telemetry Removal
*   **Sentry SDK Removal:** Stripped all Sentry.io Gradle plugins, dependency configurations, and metadata elements from Android, and deleted the Sentry framework SPM dependencies and upload build phases from iOS.
*   **Local-Only Logs:** Cleaned [CloudKitLogManager.swift](file:///Users/robertpalmer/VigilantEar/apple/VigilantEar/VigilantEar/Sources/Logging/CloudKitLogManager.swift) and Kotlin engine files to log strictly to standard local buffers (`Logger` on Apple / `Log.w` on Android).
*   **Predictive Back Gesture:** Added `enableOnBackInvokedCallback="true"` to Android Manifest to support native predictive back animations.

### 5. Multi-Platform Dynamic Thermal Throttling
*   **iOS Pipeline Duty Cycle Reduction:** Hooked into `ProcessInfo.processInfo.thermalState` inside the `AcousticProcessingPipeline` to dynamically throttle raw audio frame analysis during elevated system heat. The ML analyzer duty cycle drops from 50% (nominal) to 37.5% (serious state) and 12.5% (critical state). If critical state is reached, the pipeline injects a system warning alert (`thermal_throttle`) to the user.
*   **Android ML Interdiction:** Integrated `PowerManager.currentThermalStatus` in `AcousticEngine`. Under severe, critical, emergency, or shutdown thermal states, the engine suspends ML inference runs completely (`runInferenceWindow`) and injects a warning alert (`thermal_throttle`) while keeping native audio recording active.
*   **Tactical HUD State Integration:** Hooked up localized HUD UI indicators on both Android and iOS that subscribe to system thermal notifications to output high-visibility status labels on the radar telemetry overlay.
