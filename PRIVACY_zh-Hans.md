# Vigilant Ear 隐私政策 👂🛰️

**生效日期：** 2026 年 7 月 11 日

## 引言

Vigilant Ear（“我们”或“我们的”）致力于保护您的隐私。本隐私政策说明应用会处理哪些信息、哪些信息留在您的设备上，以及何时可能通过互联网发送有限数据以提供特定功能。

## 隐私一览

- **核心声学检测在您的设备上运行。** 声音分类、方向跟踪、实时字幕与提醒逻辑设计为使用手机的麦克风与传感器在本地工作。
- **我们不出售您的数据**，也不使用广告或行为分析 SDK。
- **我们不存储或上传录音。** 麦克风音频会实时处理以进行检测，并在启用时生成字幕；Vigilant Ear 不会将其保存为声音文件供日后播放或云端分析。
- **部分功能会使用互联网** —— 地图、恶劣天气数据源、可选的音乐识别、道路数据、应用商店购买、您*自有*设备之间的可选多手机网格流量，以及应用内法律页面的加载。详见下文。
- **控制权在您手中。** 您可随时禁用 Shazam 音乐识别、关闭提醒类别、保持 Constellation 关闭、在系统设置中撤销权限，或停止后台监听。

## 在您设备上处理的信息

在获得您的许可后，Vigilant Ear 会**在本地**访问以下内容：

- **麦克风音频** —— 实时用于检测环境声音（警报器、车辆、门铃、婴儿哭声、附近的人等）、估算方向，并在 Speaker Mode 开启时生成实时字幕与可选的设备端翻译。
- **语音识别（设备端）** —— 启用字幕时，设备上的语音框架会将附近语音转录为文本。字幕文本实时显示，Vigilant Ear 不会将其存档为永久转录历史；调试日志不包含字幕内容。
- **位置** —— 用于在地图上放置检测到的声音与天气警报区域，并改进方向指引。
- **设备方向与运动** —— 用于提高方位精度。
- **相机（可选）** —— 仅在您打开相机 AR“看见声音”视图时使用，以便在实时相机预览中钉住标记。相机帧用于设备端显示；Vigilant Ear 不会为声音识别而上传它们。
- **Apple Watch（可选）** —— 当可用 Watch 伴侣时，提醒标签与方向线索可能中继到已配对的 Watch，便于您看手腕。

这种设备端处理是应用的核心。竞品应用常将音频串流至云端进行分析与变现。Vigilant Ear 的构建方式不同：您的声学感知流水线设计为在手机本身上运行。

## 网络与第三方服务

当您使用某些功能 —— 或应用需要它们才能运行时 —— **有限数据可能离开您的设备**，并由第三方服务按其各自隐私政策处理：

*   **地图显示**
    *   *发送内容：* 地图图块请求；渲染地图所需的地图视口与大致位置
    *   *提供方：* Apple Maps / MapKit
*   **恶劣天气警报**
    *   *发送内容：* 对公共 CAP/Atom 天气数据源的请求；您的大致区域可能从数据源选择与设备位置推断
    *   *提供方：* 美国国家气象局（U.S. National Weather Service）、MeteoGate（欧洲）、中国气象局（CMA）、韩国气象厅（KMA）、与 WMO 相关的公共来源，以及类似公共警报数据源
*   **音乐识别（可选，Power Pack+）**
    *   *发送内容：* 短音频指纹 —— 绝非原始音频 —— 在检测到音乐且启用 Shazam 时（可在设置中关闭）
    *   *提供方：* Apple Shazam / ShazamKit
*   **道路上下文**
    *   *发送内容：* 基于您位置周围地图扇区的匿名 Overpass API 查询
    *   *提供方：* 通过 Overpass API 的 OpenStreetMap 贡献者
*   **购买与权益**
    *   *发送内容：* 可选一次性 Power Pack+ 解锁（非订阅）的购买令牌与权益 / 试用状态
    *   *提供方：* Apple App Store
*   **Constellation 网格（可选，Power Pack+）**
    *   *发送内容：* 当您启用多手机 Constellation 时，参与设备会交换形成共享画面所需的声学元数据 —— 例如相对姿态 / 可用时的 Ultra-Wideband 测距、方位、声音标签与短暂字幕文本。流量在您关联的手机之间点对点进行；Wingdings 不为此音频流水线运营云端网格中继。
    *   *提供方：* 您设备之间的 Apple 框架（例如 Network / Nearby Interaction）
*   **应用内法律文档**
    *   *发送内容：* 当您在应用中打开隐私政策、条款、支持或产品 README 页面时的标准网络请求
    *   *提供方：* GitHub（文档托管）

我们选择这些服务以提供地图、天气、音乐标签、购买与多设备功能。**Wingdings 不会从这些提供方接收您的麦克风音频、连续位置历史或联系信息。**

## Wingdings 收集的内容

### 无远程遥测或诊断

Vigilant Ear 设计为完全在您的设备上本地运行。我们不会在 Wingdings 服务器上收集、传输或存储远程遥测、崩溃日志、诊断记录或使用分析。可选的**本地**调试日志可能写入设备以供排查；应用不会将其作为遥测流水线上传，导出的调试内容不包含字幕文本。

## 我们不会做的事

我们**不会**：

- 出售或出租您的个人信息
- 在我们的服务器上存储环境录音
- 运行广告网络、跨应用跟踪器或行为画像 SDK
- 将您的连续位置轨迹上传至 Wingdings
- 为云端语音或声音识别上传原始麦克风音频

## 您的选择与控制

您可以：

- 在 iOS 设置中**撤销权限**（麦克风、位置、相机、通知、语音识别）
- 在 Power Pack+ / 偏好设置中**禁用 Shazam 音乐识别**
- **关闭单个提醒类别**（警报器、天气、门铃、婴儿等）
- 在所有提醒类别均禁用时**停止后台监听**
- **保持 Constellation 关闭**，从而不与其他手机共享网格元数据
- **使用 Demo Mode** 在本地预览提醒与功能，带有清晰的 DEMO 水印，且不暗示真实紧急情况

## 平台指南

Vigilant Ear 遵循 Apple App Store 隐私要求以及 Apple 面向服务无障碍需求用户之应用的指南。当我们的做法或平台义务发生变化时，我们会更新本政策。

## 政策变更

我们可能不时更新本隐私政策。重大变更将通过更新本页顶部的**生效日期**予以体现。
