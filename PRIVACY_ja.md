# Privacy Policy for Vigilant Ear 👂🛰️

**発効日：** 2026年7月11日

## はじめに

Vigilant Ear（「当社」「私たち」または「当方」）は、お客様のプライバシー保護に努めています。本プライバシーポリシーは、アプリが処理する情報、デバイス上に留まるもの、特定機能の提供のために限定的なデータがインターネット経由で送信される場合について説明します。

## プライバシーの概要

- **中核の音響検出はデバイス上で動作します。** 音の分類、方向追跡、ライブキャプション、アラートロジックは、電話のマイクとセンサーを使いローカルで動作するよう設計されています。
- **お客様のデータを販売しません。** 広告や行動分析 SDK も使用しません。
- **音声録音を保存・アップロードしません。** マイク音声は検出および（有効時）キャプションのためにリアルタイム処理され、後から再生やクラウド分析のために Vigilant Ear がサウンドファイルとして保存することはありません。
- **一部の機能はインターネットを使用します** — 地図、厳しい天候フィード、任意の音楽識別、道路データ、App Store 購入、任意の複数電話メッシュ通信（*お客様の*デバイス間）、アプリ内法務ページの読み込み。以下に説明します。
- **管理はお客様にあります。** Shazam 音楽識別の無効化、アラートカテゴリのオフ、Constellation のオフ、システム設定での権限取り消し、バックグラウンド聴取の停止をいつでも行えます。

## デバイス上で処理される情報

お客様の許可により、Vigilant Ear は次の情報に**ローカルで**アクセスします：

- **マイク音声** — 環境音（サイレン、車両、ドアベル、赤ちゃんの泣き声、近くの人など）の検出、方向の推定、（Speaker Mode オン時）ライブキャプションと任意のオンデバイス翻訳のためにリアルタイムで使用されます。
- **音声認識（オンデバイス）** — キャプション有効時、デバイスの音声フレームワークが近くの発話を電話上でテキストに書き起こします。キャプションテキストはライブ表示され、Vigilant Ear が恒久的なトランスクリプト履歴としてアーカイブすることはありません。デバッグログにキャプション内容は含まれません。
- **位置情報** — 検出された音と天気アラート区域を地図上に配置し、方向ガイダンスを改善するために使用されます。
- **デバイスの向きとモーション** — 方位精度の向上に使用されます。
- **カメラ（任意）** — カメラ AR の「音を見る」ビューを開いた場合のみ使用され、ライブカメラプレビューにマーカーを固定します。カメラフレームはオンデバイス表示用であり、Vigilant Ear が音認識のためにアップロードすることはありません。
- **Apple Watch（任意）** — Watch コンパニオンが利用可能な場合、アラートラベルと方向の手がかりがペアの Watch に中継され、手首を見て確認できます。

このオンデバイス処理がアプリの中核です。競合アプリの多くは分析と収益化のために音声をクラウドへストリーミングします。Vigilant Ear は異なります：音響認識パイプラインは電話自体で動くよう設計されています。

## ネットワークとサードパーティサービス

特定の機能を使う場合、またはアプリの動作に必要な場合、**限定的なデータがデバイスから送信**され、各社のプライバシーポリシーに従ってサードパーティサービスが処理することがあります：

*   **地図表示**
    *   *送信されるもの：* 地図タイル要求；地図の描画に必要なビューポートとおおよその位置
    *   *提供者：* Apple Maps / MapKit
*   **厳しい天候アラート**
    *   *送信されるもの：* 公開 CAP/Atom 天気フィードへの要求；フィード選択とデバイス位置から一般的な地域が推測される場合があります
    *   *提供者：* U.S. National Weather Service、MeteoGate（欧州）、China Meteorological Administration（CMA）、Korea Meteorological Administration（KMA）、WMO 関連の公開ソース、および類似の公開アラートフィード
*   **音楽識別（任意、Power Pack+）**
    *   *送信されるもの：* 短い音声フィンガープリント — 生の音声は送られません — 音楽が検出され Shazam が有効な場合（設定でオフにできます）
    *   *提供者：* Apple Shazam / ShazamKit
*   **道路コンテキスト**
    *   *送信されるもの：* 位置周辺の地図セクタに基づく匿名の Overpass API クエリ
    *   *提供者：* Overpass API 経由の OpenStreetMap コントリビューター
*   **購入と権利**
    *   *送信されるもの：* 任意の一回限りの Power Pack+ アンロック（サブスクリプションではありません）の購入トークンおよび権利 / トライアル状態
    *   *提供者：* Apple App Store
*   **Constellation メッシュ（任意、Power Pack+）**
    *   *送信されるもの：* 複数電話の Constellation を有効にすると、参加デバイスが共有の像に必要な音響メタデータを交換します — 例：相対姿勢 / 利用可能な場合の Ultra-Wideband 測距、方位、音ラベル、一時的なキャプションテキスト。通信はお客様がリンクした電話間のピアツーピアです。Wingdings はこの音声パイプライン用のクラウドメッシュ中継を運営していません。
    *   *提供者：* お客様のデバイス間の Apple フレームワーク（例：Network / Nearby Interaction）
*   **アプリ内法務文書**
    *   *送信されるもの：* アプリ内でプライバシーポリシー、利用規約、サポート、製品 README ページを開いたときの標準的なウェブ要求
    *   *提供者：* GitHub（文書ホスティング）

これらのサービスは、地図、天候、音楽ラベル、購入、複数デバイス機能を提供するために選択しています。**Wingdings は、これらの提供者からマイク音声、継続的な位置履歴、または連絡先情報を受け取りません。**

## Wingdings が収集するもの

### リモートのテレメトリや診断なし

Vigilant Ear はデバイス上で完全にローカルに動作するよう設計されています。Wingdings のサーバー上でリモートテレメトリ、クラッシュログ、診断記録、利用分析を収集・送信・保存することはありません。任意の**ローカル**デバッグログがトラブルシューティング用にデバイスへ書き込まれることがありますが、テレメトリパイプラインとしてアプリがアップロードすることはなく、キャプションテキストはエクスポートされるデバッグ内容に含まれません。

## 行わないこと

当社は次のことを**行いません**：

- 個人情報の販売または賃貸
- 環境音声録音を当社サーバーに保存すること
- 広告ネットワーク、クロスアプリトラッカー、行動プロファイリング SDK の運用
- 継続的な位置の軌跡を Wingdings へアップロードすること
- クラウド音声認識や音認識のために生のマイク音声をアップロードすること

## お客様の選択と管理

次のことができます：

- iOS 設定で**権限を取り消す**（マイク、位置情報、カメラ、通知、音声認識）
- Power Pack+ / 設定で **Shazam 音楽識別を無効にする**
- **個別のアラートカテゴリをオフにする**（サイレン、天候、ドアベル、赤ちゃんなど）
- すべてのアラートカテゴリを無効にした場合の**バックグラウンド聴取の停止**
- **Constellation をオフのままにする**ことで、他の電話とメッシュメタデータを共有しない
- **Demo Mode** を使い、明確な DEMO 透かし付きでアラートと機能をローカルにプレビューし、実際の緊急事態であるかのように装わない
